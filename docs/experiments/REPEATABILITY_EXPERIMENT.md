# CYVX Repeatability Experiment

## Goal
Prove CYVX can repeatedly observe, predict, improve, and learn from a real repository.

## Baseline
Repository: d46382015-netizen/CYVXAI-OS
Current health: 84.667/100
Status: healthy
Open issues: 0
Open pull requests: 0
Build: passing

## Experiment
Run the same proof loop after each real repository change for 7 days.

## Evidence Collected
- repository health score
- build status
- workflow failure count
- open issue count
- open PR count
- prediction error
- trust score
- intervention classification

## Success Criteria
- health remains above 80
- build remains passing
- prediction error decreases or stays stable
- proof ledger records multiple before/after deltas
- next best actions become more accurate

## Next Bottleneck
Move from single-repo proof to repeated proof and then second-repo proof.
